# Color nodes by module

If you have module assignments — one integer per ROI from a community-
detection algorithm — the simplest way to color nodes is `--node-color module`:

```bash
hlplot plot ... \
  --modules k5_state_0/module_assignments.csv \
  --node-color module
```

Each module gets a distinct color from a perceptually-uniform palette
generated by `generate_module_colors`. The palette uses HSL rotation so
colors stay distinct even at 10+ modules.

## Edges colored by module too

To extend module coloring to edges, switch to `hlplot modular`:

```bash
hlplot modular ... \
  --modules k5_state_0/module_assignments.csv \
  --edge-color-mode module
```

Intra-module edges adopt the module color; inter-module edges render in
muted gray. See [Modularity tutorial](../tutorials/modularity.md) for the
full pipeline.

## Custom colors per node

If you want hand-picked colors (e.g. matching a paper's color scheme), pass
a CSV of hex codes:

```bash
hlplot plot ... --node-color file:custom_node_colors.csv
```

The CSV must have one row per ROI in the same order as your coordinates,
with a single `color` column of hex strings (e.g. `#ff7f0e`).

## In Python

```python
from HarrisLabPlotting import generate_module_colors, create_brain_connectivity_plot
import numpy as np

modules = np.loadtxt("module_assignments.csv", dtype=int)
n_modules = int(modules.max()) + 1
palette = generate_module_colors(n_modules, seed=42)
node_colors = [palette[m] for m in modules]

fig, _ = create_brain_connectivity_plot(
    ...,
    node_color=node_colors,
)
```
